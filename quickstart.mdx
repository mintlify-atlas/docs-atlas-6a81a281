## Quick start guide

This guide will walk you through the basic workflow of using know to search your local files.

<Steps>
  <Step title="Add a directory to watch">
    First, add a directory containing files you want to search:
    
    <CodeGroup>
    
    ```bash uv
    uv run know add ~/Documents/notes
    ```
    
    ```bash pipx
    know add ~/Documents/notes
    ```
    
    </CodeGroup>
    
    You'll see confirmation:
    ```
    Added /Users/yourname/Documents/notes
    ```
    
    <Tip>
      You can add multiple directories by running the `add` command multiple times. All watched directories are tracked in `~/.know_dirs`.
    </Tip>
  </Step>
  
  <Step title="Index your files">
    Index all files in your watched directories:
    
    <CodeGroup>
    
    ```bash uv
    uv run know index
    ```
    
    ```bash pipx
    know index
    ```
    
    </CodeGroup>
    
    You'll see a progress bar as know processes your files:
    ```
    Indexing 1 directories
    Indexing ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 100% 12/12 0:00:03 added 342 · skipped 0
    OK Total: 342 new, 0 unchanged
    ```
    
    <Note>
      The first indexing run will process all files. Subsequent runs are incremental and only process new or modified files.
    </Note>
  </Step>
  
  <Step title="Search your files">
    Now you can search using natural language queries:
    
    <CodeGroup>
    
    ```bash uv
    uv run know "retrieval augmented generation"
    ```
    
    ```bash pipx
    know "retrieval augmented generation"
    ```
    
    </CodeGroup>
    
    You'll get ranked results with a detailed preview of the top match:

## Search examples

know supports multiple search modes and options:

### Default semantic search

Use natural language queries to find semantically similar content:

<CodeGroup>

```bash Semantic query
know "how to implement authentication"
```

```bash Technical concepts
know "database connection pooling"
```

```bash Code patterns
know "async error handling patterns"
```

</CodeGroup>

### BM25 lexical search

Use `--bm25` for keyword-based search when you need exact term matching:

```bash
know "chromadb persist" --bm25
```

<Note>
  BM25 search builds a lightweight index from stored chunks and caches it under `know_index/bm25` for fast subsequent queries.
</Note>

### Hybrid search

Combine both approaches with `--hybrid` for the best of both worlds:

```bash
know "vector embeddings" --hybrid
```

Hybrid mode uses Reciprocal Rank Fusion (RRF) to merge results from both dense vector search and BM25 lexical search:

```python Fusion algorithm (src/retrieval.py:19-32)
def rrf_fuse(
    result_lists: Iterable[list[SearchItem]], k: int = 60, limit: int = 5
) -> list[FusedItem]:
    scores: dict[str, dict] = {}
    for items in result_lists:
        for rank, item in enumerate(items, 1):
            entry = scores.get(item.key)
            if entry is None:
                scores[item.key] = {"item": item, "score": 0.0}
            scores[item.key]["score"] += 1.0 / (k + rank)

    fused = [FusedItem(v["item"], v["score"]) for v in scores.values()]
    fused.sort(key=lambda x: x.score, reverse=True)
    return fused[:limit]
```

### Limit results

Control the number of results with `--limit`:

```bash
know "python testing" --limit 10
```

### Filter by file patterns

Use `--glob` to search only specific file patterns:

```bash
know "api endpoints" --glob "**/*.py"
know "documentation" --glob "notes/**" --glob "docs/**/*.md"
```

### Filter by modification time

Use `--since` to search only recently modified files:

<CodeGroup>

```bash Last 7 days
know "bug fixes" --since 7d
```

```bash Last 12 hours
know "recent changes" --since 12h
```

```bash Specific date
know "Q1 planning" --since 2024-01-15
```

</CodeGroup>

## Managing directories

### View watched directories

See all directories currently being watched:

```bash
know dirs
```

Output:

### Remove a directory

Stop watching a directory:

```bash
know remove ~/Documents/notes
```

<Warning>
  Removing a directory from the watch list doesn't delete its chunks from the index. Use `know prune` to clean up orphaned chunks.
</Warning>

### Clean up orphaned chunks

Remove chunks from files that have been deleted:

```bash
know prune
```

Preview what would be removed with `--dry`:

```bash
know prune --dry
```

Output:
```
Dry run: Would remove 47 orphaned chunks (from 3 deleted files)
```

## Advanced indexing options

### Index specific file types

Focus on specific file extensions for faster indexing:

```bash
know index --ext .md --ext .txt
```

### Index with glob patterns

Only index files matching specific patterns:

```bash
know index --glob "**/*.py" --glob "docs/**"
```

### Index recent files only

Skip older files when indexing:

```bash
know index --since 7d
```

### Customize chunking

Adjust chunk size and overlap for your use case:

```bash
know index --chunk-size 512 --overlap 50
```

<Note>
  Default chunk size is 512 tokens with 50 tokens overlap. Larger chunks provide more context but may reduce precision. Smaller chunks improve precision but may lose context.
</Note>

### Force rebuild

Clear and rebuild the entire index:

```bash
know index --force
```

<Warning>
  `--force` deletes all existing chunks and reindexes everything from scratch. This can take significant time for large document sets.
</Warning>

### Dry run

Preview what would be indexed without making changes:

```bash
know index --dry
```

Output:
```
Dry run: Scanning 1 directories
Dry run: Would index 342 new, 0 unchanged
```

### Debug indexing

See detailed logs of what's being scanned and chunked:

```bash
know index --log
```

## Output formats

### Plain text output

Get simple text output without rich formatting:

```bash
know "search query" --plain
```

### JSON output

Get results in JSON format for scripting:

```bash
know "search query" --json
```

Save JSON to a file:

```bash
know "search query" --json-out results.json
```

## Benchmark mode

Compare dense and BM25 search results side-by-side:

```bash
know "search query" --benchmark
```

This displays both result sets so you can evaluate which search mode works better for your query.

## Command shortcuts

know automatically prefixes unrecognized commands with `search`, so these are equivalent:

<CodeGroup>

```bash Explicit
know search "your query"
```

```bash Shortcut
know "your query"
```

</CodeGroup>

## Tips and best practices

<AccordionGroup>
  <Accordion title="When to use each search mode" icon="compass">
    - **Dense (default)**: Best for semantic queries and conceptual search
    - **BM25**: Best for exact keyword matching and technical terms
    - **Hybrid**: Best when you want both semantic understanding and keyword precision
  </Accordion>
  
  <Accordion title="Optimizing index performance" icon="gauge-high">
    - Use `--ext` to limit file types and speed up indexing
    - Use `--glob` to focus on specific directories
    - Use `--since` to skip unchanged files
    - Run `know prune` periodically to remove orphaned chunks
    - Index incrementally by running `know index` regularly
  </Accordion>
  
  <Accordion title="Managing large document sets" icon="database">
    - Start with smaller directories to test
    - Use `--dry` to preview indexing before committing
    - Monitor chunk counts with `know dirs`
    - Adjust `--chunk-size` based on your document types
    - Use `--force` sparingly as it rebuilds everything
  </Accordion>
  
  <Accordion title="Debugging search results" icon="bug">
    - Use `--benchmark` to compare search modes
    - Try different chunk sizes if results are poor
    - Use `--log` when indexing to see what's being processed
    - Check file modification times with `--since` filters
    - Verify glob patterns match expected files
  </Accordion>
</AccordionGroup>
